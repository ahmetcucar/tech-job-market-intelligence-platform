# Learning Context

This file is a working note for how to guide the project build. It is not part of the public product pitch. It exists so explanations can match the builder's current background, strengths, and learning goals.

## Background

I have about 2 years of data engineering experience.

I know some data engineering concepts, but I am still growing and do not want to assume I know everything.

## Comfortable Areas

I am most comfortable with:

- SQL
- ETL and ELT concepts
- Data pipelines
- Basic data modeling
- Python for data work
- Data warehouse concepts
- Batch processing concepts
- Thinking about source data, transformations, and analytics outputs

## Growth Areas

I am still learning:

- Production backend engineering
- API design
- Frontend development
- Authentication
- Async workers
- Deployment
- Software testing patterns
- Deeper system design
- Performance optimization outside of analytics workloads
- CI/CD

## Beginner Areas

I am a complete beginner in:

- AI engineering
- Embeddings
- Vector databases
- RAG
- LLM evaluation
- Prompt pipelines
- Recommendation systems
- AI product design

## How To Teach Me

When explaining software engineering or AI engineering topics:

- Explain like I am new to the topic.
- Define unfamiliar terms.
- Use small examples before introducing larger patterns.
- Connect concepts back to data engineering when possible.
- Explain why each technical choice is being made.
- Avoid assuming I already know backend, frontend, or AI patterns.
- Break complex tasks into step-by-step implementation plans.

When explaining data engineering topics:

- Do not over-explain the basics unless needed.
- Still call out production best practices and tradeoffs.
- Help me strengthen gaps around orchestration, modeling, observability, reliability, and scale.

## Project Learning Goal

Use this project to grow from a junior data engineer into someone who can credibly build modern data, backend, and AI systems end to end.

The project should build confidence in:

- Designing complete systems
- Explaining architecture decisions
- Building backend services
- Adding AI features responsibly
- Writing clean project documentation
- Turning technical work into strong resume and interview stories
